SASS Variables & maps

Variables

First very useful SASS feature are variables. We can defined them on the top of our file, and re-use in multiple places:

 
    $mycolor: #1d61aff4;

    body {
        background-color: $mycolor;
    }

Obviously now CSS also offers us variables (so called custom properties) but taking into account other features which are offered by SASS it's still more convenient to use SASS. I also personally feel that SASS variables are more intuitive than CSS ones.

As you can imagine, this is very handy tool which allows us to easily update colors in entire page by changing just a single value.

Most of the pages use some theme color and defines default color of

  • primary color
  • secondary color
  • background color
  • accent color

I.e. when you check Bootstrap source, you will find .scss files and variables responsible for colors, which you can overwrite:


    // Your variable overrides
    $body-bg: #000;
    $body-color: #111;
    
    // Bootstrap and its default variables
    @import "../node_modules/bootstrap/scss/bootstrap";

Once we save the file, compiler will compile our variables into final css code. As you can see variables are removed from output css file.

Obviously we can use variable for other properties as well: 


    $mycolor: #1d61aff4;
    $font-stack: 'Courier New', Courier, monospace;
    
    body {
        background-color: map-get($colors, background);
        font-family: $font-stack;
    }

SASS maps

Similar to JavaScript objects 


    $colors: (
    "primary": #1d61aff4,
    "accent": #accc20,
    "background": #dddddd
);

In order to apply them we have to slightly change syntax:


    body {
        background-color: map-get($colors, background);
    }

Maps are also available in the Bootstrap: 


    $theme-colors: (
        "primary": #0074d9,
        "danger": #ff4136
      );
