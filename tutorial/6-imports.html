Imports

When it comes to complex project it's difficult to work on a single file.

SASS allows us to easily split our code into multiple sub-files and merge them during compilation time:

Let's create new file called _footer.scss and add the following code:


    .footer {
        height: 50px;
        background-color: white;
    }

Within the main styles.scss file add the following import statement:


    .footer {
        height: 20px;
        background-color: white;
        position: fixed;
        left: 0;
        bottom: 0;
        width: 100%;
        text-align: center;
    }

Now when we check our output file (styles.css) you will see that our footer styles has been added so the last thing we have to do is add our footer to the HTML:


    <div class="footer">Copyrights @MDBootstrap.com</div>

Variables

It's a common practice to keep all variables in a separate file, let's move them to _variables.scss file and import in styles.scss

Extend

Very often we end up creating classes which are very similar to each other and have only one or two different values. Instead of retyping all parameters (and adjusting all of the in we decide to change some common attribute) we can use @extend option:


    .subtitle{
        @include flexAlign();
        font-size: 30px;
    }

    .quote {
        @extend .subtitle;
        font-size: 20px;
    }

This will create a class called .quote which will have exactly the same properties as .subtitle class but smaller font-size.
